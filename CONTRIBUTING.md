# Contributing

Thanks for helping improve 灵墟夜行.

## Local Development

```bash
npm install
npm run dev
```

Before opening a pull request, run:

```bash
npm run build
```

## Pull Requests

- Keep changes focused and easy to review.
- Include a short description of gameplay or UI behavior changes.
- For new assets, include source notes and confirm they can be distributed with the project.
- Avoid committing generated build output from `dist/`.

## Issue Ideas

Good first issues include balancing card values, adding new cards, improving mobile controls, and writing small tests around pure game logic after it is extracted from the Phaser scene.
